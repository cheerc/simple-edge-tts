# simple-edge-tts v0.1.0: 使用者驗證清單

> Generated: 2026-06-24
> Plan: `docs/plan/2026-06-24-implementation.md`
> Spec: `docs/spec/2026-06-24-design.md`

本清單供**人工驗證**第一次開發 Loop 產出的程式。分為兩大階段：開發環境驗證（`python -m src.main`）和打包後驗證（.exe / .dmg）。

---

## How to test（開發環境）

```bash
cd simple-edge-tts
pip install -e ".[dev]"
python -m src.main
```

## §3: 語言切換（i18n）

### 驗證步驟

## §5: 參數調整（Rate / Pitch）

### 驗證步驟
1. 拖動語速滑桿 → 觀察數值標籤
2. 拖動音調滑桿 → 觀察數值標籤
3. 雙擊滑桿 → 重設為預設值
4. 調整後關閉重開 app

### 預期行為
- Rate 範圍 -50% ~ +100%，預設 0%
- Pitch 範圍 -50Hz ~ +50Hz，預設 0Hz
- 數值標籤即時更新（如 `+20%`）
- 雙擊重設
- 參數跨 session 保存

## §6: TTS 試聽（需網路）

### 驗證步驟
1. 輸入文字 → 點「▶ 試聽」
2. 聽播放
3. 播放中點「⏹ 停止」
4. 清空文字 → 觀察按鈕狀態

### 預期行為
- 生成中顯示進度指示（loading）
- 音訊從喇叭播出
- 播放中按鈕變為「⏹ 停止」
- 停止後可再次試聽
- 文字為空時按鈕 disabled

## §8: 錯誤 / Edge Case

### 驗證步驟
1. 斷開網路 → 嘗試試聽/匯出
2. 輸入超長文字（5000+ 字）→ 匯出
3. 輸入特殊字元（emoji 🎵、換行、HTML tag）→ 試聽

### 預期行為
- 無網路：顯示錯誤訊息，不 crash
- 長文字：正常處理（可能較慢）
- 特殊字元：不 crash，合理處理

## §9: 自動更新通知

### 驗證步驟
1. 啟動 app → 觀察是否有更新通知
2. [可選] 手動修改版號為 `0.0.1` → 啟動 → 應顯示更新通知

### 預期行為
- 啟動時背景檢查 GitHub Releases API
- 有新版 → 顯示非阻塞通知（不擋操作）
- 無新版 → 不顯示任何東西
- 網路錯誤 → 靜默失敗，不打擾使用者
